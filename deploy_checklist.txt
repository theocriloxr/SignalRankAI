SignalRankAI Deployment & Automation Checklist

1. Environment Setup
- Copy .env.example to .env and fill in secrets (TELEGRAM_TOKEN, PAYSTACK_SECRET_KEY, DATABASE_URL)
- Add .env to .gitignore

2. Dependency Installation
- pip install -r requirements.txt

3. Process Management
- Use start.sh for local multi-process launch
- Use Procfile for Heroku/Railway deployment
- Use railway.json for Railway cloud deployment

4. Bot & Engine Startup
- Recommended (canonical):
	- RUN_MODE=bot python main.py (Telegram bot)
	- RUN_MODE=engine python main.py (Signal engine loop)
	- RUN_MODE=web python main.py (FastAPI web / health / metrics / webhooks)
	- RUN_MODE=worker python main.py (subscription expiry + background jobs)

4b. Railway (production) layout options

- Create 1 Railway project
- Add Postgres plugin

- Option A: Single service (all-in-one)
	- Create one service named e.g. "all" or set RUN_MODE=all
	- Start command: `python main.py`
	- Runs web + bot + engine + worker in one container via threads
	- Keep in mind memory/CPU; async DB pool uses NullPool in RUN_MODE=all to avoid cross-loop issues

- Option B: Four services (separated)
	- Create 4 services (same repo) with start command: `python main.py`
	- Name services so RUN_MODE is auto-detected: "web", "bot" (or "telegram"), "engine", "worker"
	- RUN_MODE is still honored if set manually; auto-detect just makes Railway zero-config per service

- Configure variables on ALL services:
	- DATABASE_URL (from Railway Postgres)
	- TELEGRAM_TOKEN
	- PAYSTACK_SECRET_KEY
	- PAYSTACK_WEBHOOK_SECRET (or reuse PAYSTACK_SECRET_KEY)
	- OWNER_TELEGRAM_ID (or OWNER_IDS)
	- BYPASS_KEY
	- ML_TRAIN_ENABLED=true (worker only; optional, defaults true)
	- ML_TRAIN_INTERVAL_SECONDS=86400 (worker only; optional)
	- AUTO_MIGRATE=true (default); DB_CONNECT_MAX_ATTEMPTS / DB_CONNECT_BACKOFF_SECONDS for cold starts
    - ULTRA_QUALITY_ENABLED=false (disable ultra-strict filter that requires many fields; enable only after enriching signals)

- Data Source Configuration (Multi-Provider Support):

	Crypto Data Providers (priority order):
		- CRYPTO_DATA_PROVIDER=binance (default; options: binance, bybit, cryptocompare)
		- CRYPTOCOMPARE_API_KEY (optional; free tier available)
    
	FX Data Providers (priority order):
		- OANDA_API_KEY (free demo account; bank-grade data)
		- OANDA_ACCOUNT_ID (for OANDA)
		- OANDA_PRACTICE=true (default; use practice/demo account)
		- ALPHAVANTAGE_API_KEY (legacy; limited free tier)
    
	Stock Data Providers:
		- No API key needed for Yahoo Finance (default free provider)
		- POLYGON_API_KEY (optional; premium multi-asset data)
		- TWELVEDATA_API_KEY (optional; generous free tier)
    
	Universal Settings:
		- USE_MULTI_PROVIDER_DATA=true (default; enable fallback to multiple providers)
		- STOCK_TRADING_ENABLED=true (enable stock signals)
		- STOCK_TICKERS="AAPL,MSFT,GOOGL,AMZN" (manual stock list; optional)
		- STOCK_TRENDING_TOP_N=20 (auto-discover top stocks; default 20)
	- rate limiting

- Optional (dangerous): set FRESH_START=true to wipe Postgres data once on next web boot.

- Legacy (backward compatible, opt-in):
	- ALLOW_LEGACY_TELEGRAM_BOT=true python -m telegram.bot

5. Payment & Monetization
- Ensure Paystack webhook is set up and points to your server
- Test /upgrade and /subscribe flows
- Confirm subscription expiry and tier gating

6. Automation & Scaling
- Use supervisor, pm2, or systemd for process monitoring (optional)
- Set up log rotation for error and activity logs
- Schedule regular backups of database
- Monitor uptime and error rates

7. Security
- Rotate secrets regularly
- Restrict access to .env and database files
- Enable HTTPS for any web endpoints

8. Launch & Growth
- Register commands with BotFather
- Set bot description, about, and profile photo
- Share bot link on social media and groups
- Monitor user feedback and iterate

9. Advanced Automation (Optional)
- Add CI/CD pipeline for auto-deploy on git push
- Integrate with monitoring tools (UptimeRobot, Grafana)
- Schedule daily/weekly performance reports

10. Post-Launch
- Track user growth and retention
- Review logs for errors and abuse
- Plan next ML/data upgrades

Ready to deploy and automate. Follow this checklist for a smooth launch and scale.
